# GitHub Project Board Setup

Create a GitHub Project board to track delivery work for CDRPS.

## Suggested Columns

- To Do - planned work that has not started
- In Progress - active implementation work
- In Review - pull requests waiting for review
- Done - completed work that has been merged or closed

## Suggested Automation

- Move items to In Review when a pull request is opened.
- Move items to Done when a pull request is merged.
- Move issues to In Progress when work begins on a branch.

## Suggested Card Types

- Features
- Bugs
- Documentation
- Performance improvements
- Deployment tasks

## Practical Setup Tips

- Keep one issue per card.
- Write clear acceptance criteria.
- Link pull requests to the relevant issue.
- Review the board weekly to keep it current.
